Reinforcement Learning · The Deadly Triad

Baird's Counterexample

Why TD(0) with linear function approximation diverges off-policy — even with zero rewards, only 7 states, and deterministic transitions.

① Function Approximation ② Bootstrapping (TD) ③ Off-Policy Sampling
Step 01

The 7-State MDP

Baird's example uses the simplest possible MDP that exposes instability: 7 states, zero reward everywhere, and two actions per state — each with deterministic transitions.

  • Solid action → self-loop (stay in current state)
  • Dashed action → always go to s₇

The stochasticity is in the policy, not the environment. P(s'|s,a) is fully deterministic — only the action selection is random.

  • s₁–s₆ — "upper" states, heavily sampled by behavior policy
  • s₇ — "lower" state, where the target policy always goes
All rewards = 0, so V*(s) = 0 everywhere. Any divergence is purely algorithmic — not caused by a hard optimization problem.
Step 02

Two Different Policies

Behavior policy b — generates the data:

  • From upper state sᵢ: solid (self-loop) w.p. 6/7, dashed (→ s₇) w.p. 1/7
  • From s₇: solid (→ random upper state) w.p. 6/7, dashed (self-loop) w.p. 1/7

Target policy π — what we want to evaluate:

  • From upper state sᵢ: always dashed → s₇
  • From s₇: always dashed → self-loop at s₇
Under b, there is a cycle: upper states occasionally go to s₇, and s₇ occasionally bounces back to a random upper state. This spreads b's stationary distribution across all 7 states.
This mismatch IS the off-policy problem. We use data from b but update toward targets computed under π — lethal combined with FA + bootstrapping.

Step 03

Linear Function Approximation

Instead of storing one value per state, we use V(s) ≈ φ(s)ᵀ w — features φ(s) dotted with a shared weight vector w.

This is the "function approximation" component. It saves memory but couples all state estimates — updating one state's value implicitly changes others.

7 states, only 8 weights. Shared structure prevents independent estimates. The coupling through w₈ is what makes everything interact destructively.

Step 04

The TD(0) Semi-Gradient Update

We want to evaluate Vπ — the value function of π — without following π. Instead we use b to visit states, but evaluate next-state values using π's transitions.

δ  =  R + γ · V(s')  −  V(s)    // R = 0 always
w  ←  w  +  α · δ · φ(s)     // push V(s) toward γ·V(s')

s is sampled from b's distribution (upper states, frequently).
s' is the next state under π — from any upper state, π always goes to s₇.

So each update uses s from b but s' from π. These come from two different distributions — this mismatch is the off-policy problem.
The bootstrap target γ·V(s') uses current w — so as w grows, the target grows too. There is no fixed point to converge to.

Step 06

Why the Weights Diverge: The Feedback Loop

Phase 1 — Frequent Updates

Upper states sampled constantly

b visits s₁–s₆ 6/7 of the time. Each update uses s' = s₇ (π's next state from any upper state), so V(sᵢ) is repeatedly pushed toward γ·V(s₇).

Phase 2 — Bootstrapping Amplifies

Target moves with the weights

As V(sᵢ) is pushed up, V(s₇) also rises (shared w₈). This raises the bootstrap target γ·V(s₇), which pushes V(sᵢ) up further. A runaway feedback loop.

Phase 3 — Feature Coupling

w₈ ties everything together

Every upper-state feature has +1 in position 8. s₇ has +2 in position 8. Updating any upper state shifts w₈, which directly changes V(s₇) = w₇ + 2w₈.

The root cause: s (from b) and s' (from π) come from different distributions. The expected update matrix A = 𝔼b[φ(s)(φ(s) − γφ(s'))ᵀ] is not positive definite — TD(0) has no stable fixed point. With linear FA, this means divergence.
Step 07

Key Takeaways

A 7-state MDP with zero reward exposes a fundamental impossibility for a broad class of RL algorithms.

⚠️

Not a bug

Divergence is a provable property of TD(0) + FA + off-policy. Not noise, bad luck, or bad code.

📐

Linear = enough

No deep networks needed. 8 linear weights in a 7-state MDP is sufficient for divergence.

🔗

All three needed

Remove any one — go on-policy, use MC, or use tabular — and instability collapses.

🎯

True value = 0

All rewards zero. Algorithm diverges not from a hard problem but from a structural flaw.

🛡️

Fixes exist

Gradient TD, Emphatic TD, and fitted-value methods (DQN replay + target nets) address stability.

🔬

Deep RL too

DQN's experience replay + frozen target network both implicitly fight the deadly triad.

Sutton & Barto, Chapter 11

The Deadly Triad — All Three Required for Instability

① Function Approx.

Generalizes across states but couples estimates through shared weights.

② Bootstrapping

Uses current estimates as targets (TD, Q-learning) — targets move as weights change.

③ Off-Policy

Data distribution (b) ≠ target policy (π) — destroys the restoring force of on-policy learning.
